# v0.3.0 Implementation Summary - Execution Metadata in Reports

## Release Overview

DataQE Framework v0.3.0 introduces **Execution Metadata** feature that adds traceability to all generated reports. Users can now easily identify which configuration file, configuration blocks, and test YAML file were used for each report.

**Release Date**: March 6, 2026
**Version**: 0.3.0
**Status**: ✅ Complete and Ready for Deployment

## What's New

### Feature: Execution Metadata in Reports

Execution metadata provides contextual information in all reports:
- **Configuration File**: Path to the config YAML used
- **Blocks Executed**: Names of all blocks that were executed
- **Test Script**: Path to the test YAML file
- **Execution Timestamp**: When the tests started (YYYY-MM-DD HH:MM:SS)

### Reports Enhanced

1. **ExecutionReport.html**
   - Collapsible metadata section (click to expand)
   - Appears after header, before summary cards
   - Styled with blue border (#3498db)
   - Uses native HTML `<details>` and `<summary>` tags

2. **ExecutionReport.csv**
   - Metadata rows at end after summary
   - Key-value pair format
   - Easy to parse and process

3. **FailedExecutionReport.html**
   - Metadata in "failed tests" view
   - Metadata in "all passed" view
   - Consistent styling with ExecutionReport.html

## Usage

### No Configuration Needed

Metadata is captured automatically. Just run normally:

```bash
# Automatic metadata in reports
dataqe-run --config config.yml

# Multi-block with metadata showing all blocks
dataqe-run --config config.yml --all-blocks

# Specific block with metadata showing that block
dataqe-run --config config.yml --block config_block_1
```

### Report Output

**HTML Reports** show collapsible metadata:
```
📋 Execution Metadata (Click to expand)
    Configuration File: config.yml
    Blocks Executed: config_block_1, config_block_2
    Test Script: tests.yml
    Execution Time: 2026-03-06 14:23:45
```

**CSV Reports** include metadata at end:
```
EXECUTION METADATA
Configuration File,config.yml
Blocks Executed,config_block_1, config_block_2
Test Script,tests.yml
Execution Timestamp,2026-03-06 14:23:45
```

## Sending HTML Reports via Email

### The HTML is Already Perfect

✅ The HTML files generated by DataQE Framework are **properly formatted and production-ready**.

The issue users experience (HTML appearing as plain text in email) is **NOT** a problem with the HTML generation - it's about **how the HTML is being sent**.

### Solution: Use Proper MIME Type

When sending via email, use `text/html` MIME type:

**✅ Correct:**
```python
from email.mime.text import MIMEText

html_part = MIMEText(html_content, "html")  # MIME type: text/html
msg.attach(html_part)
```

**❌ Wrong:**
```python
html_part = MIMEText(html_content, "plain")  # MIME type: text/plain
msg.attach(html_part)
```

See `SENDING_REPORTS_VIA_EMAIL.md` for complete examples.

## Troubleshooting

### Issue: "HTML looks like plain text in email"

**Cause**: Email sent with `text/plain` MIME type instead of `text/html`

**Solution**: See `SENDING_REPORTS_VIA_EMAIL.md` for correct SMTP code

### Issue: "CSS not rendering in email"

**Cause**: CSS might be in external `<style>` tag that email client strips

**Solution**: All CSS is inline in the HTML - should work in all clients

### Issue: "Table misaligned on mobile"

**Cause**: HTML rendering limitation

**Solution**: Save as PDF or attach original HTML file

### Issue: "Metadata section not expanding"

**Cause**: Email client doesn't support `<details>` element

**Solution**: Metadata is also visible in plain text if expanded - client just shows all content
